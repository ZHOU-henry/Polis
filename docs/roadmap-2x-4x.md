# Agora Roadmap 2.x -> 4.0

## Intent

Agora should not grow by randomly accumulating marketplace features.

It should grow by becoming more credible across four layers:

1. market conversion
2. deployment operations
3. enterprise trust and governance
4. ecosystem and hosted runtime

## Phase 2.5

Primary goal:
turn accepted engagements into observable deployment systems.

Key additions:

- deployment graph
- expansion pipeline
- builder proof
- ops health dashboard

What success looks like:

- platform users can see how one demand turns into a delivery object and then
  into expansion or remediation
- builder trust is shown through proof and operating posture, not only listings
- ops can immediately spot blocked engagements, watch-state engagements, and
  expansion candidates

## Phase 3.0

Primary goal:
turn Agora from a delivery board into a serious enterprise transaction layer.

Key additions:

- quote and commercial workflow
- verified builder tiers
- connector packs
- enterprise governance

What success looks like:

- a buyer can compare commercial and delivery shapes instead of only response copy
- builder verification and trust become explicit platform primitives
- connectors and deployment constraints become part of the product, not hidden
  implementation details
- enterprise buyers can understand how access, approval, audit, and deployment
  scope are controlled

## Phase 4.0

Primary goal:
make Agora the runtime and ecosystem layer for third-party agent delivery.

Key additions:

- hosted agent execution
- real runtime management
- partner ecosystem
- external builder operating model

What success looks like:

- Agora is not only matching supply and demand
- it is hosting, routing, monitoring, and governing agent systems
- outside builders can bring supply into a real partner ecosystem
