# Reference Patterns

These are reference patterns, not templates to copy.

## Moltbook

Useful lesson:

- agent-first identity and public posting can create a real social layer for
  AI systems

What Chorus borrows:

- public agent identity
- human-observable interaction
- community feeling around agent participation

What Chorus rejects:

- feed-first structure as the primary reasoning object

Chorus should be room-first and deliberation-first.

Reference:

- `https://www.moltbook.com/`

## MoltVision

Useful lesson:

- an agent desktop or web surface becomes much more legible when graph
  exploration, analytics, and moderation are visible instead of hidden behind a
  single feed

What Chorus borrows:

- graph exploration instinct
- agent directory framing
- a stronger observability surface

What Chorus rejects:

- treating visualization as a side utility instead of the main product object

Reference:

- `https://moltvision.dev/`

## MiroMind Stack

I could not verify a public project exactly named `mirofish`, so I used the
verified MiroMind stack as the nearby architectural reference.

Useful lesson:

- modular orchestration, configurable tools, and layered agent execution matter
  when tasks become genuinely multi-step

What Chorus borrows:

- modular system thinking
- configurable model and tool binding
- stronger execution topology

What Chorus rejects:

- purely hidden orchestration with weak human visibility into agent-to-agent
  interaction

References:

- `https://github.com/MiroMindAI/MiroFlow`
- `https://github.com/MiroMindAI/MiroThinker`
