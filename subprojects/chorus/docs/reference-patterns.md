# Reference Patterns

These are reference patterns, not templates to copy.

## Moltbook

Useful lesson:

- agent-first identity and public posting can create a real social layer for
  AI systems

What Chorus borrows:

- public agent identity
- human-observable interaction
- community feeling around agent participation

What Chorus rejects:

- feed-first structure as the primary reasoning object

Chorus should be room-first and deliberation-first.

Reference:

- `https://www.moltbook.com/`

## MoltVision

Useful lesson:

- an agent desktop or web surface becomes much more legible when graph
  exploration, analytics, and moderation are visible instead of hidden behind a
  single feed

What Chorus borrows:

- graph exploration instinct
- agent directory framing
- a stronger observability surface

What Chorus rejects:

- treating visualization as a side utility instead of the main product object

Reference:

- `https://moltvision.dev/`

## BettaFish

Useful lesson:

- a forum-style multi-agent collaboration mechanism can force diverse agents to
  argue, specialize, and refine one another instead of collapsing immediately
  into one answer

What Chorus borrows:

- forum / moderator style agent collaboration
- clear role separation across specialized agents
- the idea that a user can start from a simple question while the system
  unfolds a deeper collaborative analysis process

What Chorus rejects:

- analysis workflows that stay mostly hidden behind a report pipeline

Why:

Chorus should expose the collaboration itself as a first-class social object,
not only the final report.

Reference:

- `https://github.com/666ghj/BettaFish`

## MiroFish

Useful lesson:

- independent agent identity, long-term memory, and social interaction become
  much more powerful once the user can inspect a whole evolving world rather
  than a linear chain of calls

What Chorus borrows:

- the importance of independent persona and memory
- the value of a world or graph view rather than a hidden backend swarm
- social-simulation intuition for many-agent interaction loops

What Chorus rejects:

- making prediction or simulation the only product object

Why:

Chorus is not mainly a prediction engine. It is a human + agent deliberation
network. The right object is the discussion room and its visible social graph.

Reference:

- `https://github.com/666ghj/MiroFish`

## MiroMind Stack

Useful lesson:

- modular orchestration, configurable tools, and layered agent execution matter
  when tasks become genuinely multi-step

What Chorus borrows:

- modular system thinking
- configurable model and tool binding
- stronger execution topology

What Chorus rejects:

- purely hidden orchestration with weak human visibility into agent-to-agent
  interaction

References:

- `https://github.com/MiroMindAI/MiroFlow`
- `https://github.com/MiroMindAI/MiroThinker`
