# Peras Frontier Open-Project Landscape 2026

## 1. Core Research And Review Infrastructure

### OpenAlex

- Link: <https://openalex.org/>
- Core idea:
  open global graph of papers, authors, institutions, topics, and citations
- Why Peras cares:
  essential for category mapping, topic clustering, and institutional movement

### OpenReview

- Link: <https://openreview.net/>
- Core idea:
  public review and submission surface for frontier machine-learning research
- Why Peras cares:
  one of the fastest direct signals for frontier technical movement

### Papers with Code dataset archive

- Link: <https://github.com/paperswithcode/paperswithcode-data>
- Core idea:
  open archive for papers, methods, datasets, and evaluation tables
- Why Peras cares:
  useful historical benchmark and paper-to-code signal layer

## 2. Open Build And Engineering Signal

### GH Archive

- Link: <https://www.gharchive.org/>
- Core idea:
  archive the public GitHub event stream for analysis
- Why Peras cares:
  strong signal for implementation velocity and ecosystem attention

### Hugging Face Hub

- Link: <https://huggingface.co/>
- Core idea:
  distribution layer for models, datasets, and evaluation assets
- Why Peras cares:
  useful for tracking model release momentum and open ecosystem adoption

## 3. Global News And Macro Signal

### GDELT

- Link: <https://www.gdeltproject.org/>
- Core idea:
  global event and media monitoring system
- Why Peras cares:
  important for detecting geographic, political, and industry signal outside
  pure ML research channels

## 4. What The Current Landscape Says

The Peras opportunity is not to invent raw data.

The opportunity is to:

- fuse research and market signal better
- maintain explicit uncertainty
- separate high-frequency noise from meaningful directional movement
- produce ranked judgments rather than streams of disconnected updates
