# Update Cadence

## Principle

Peras should update in layers, not with one giant unscheduled dump.

## Daily Layer

Purpose:

- capture new papers
- capture major repo and company movement
- record raw signal deltas

Output:

- raw ingest update
- flagged anomalies

## Weekly Layer

Purpose:

- convert raw signal into structured summaries
- note what changed materially

Output:

- weekly frontier memo
- ranked changes in direction momentum

## Monthly Layer

Purpose:

- revise thesis cards
- compare emerging directions
- update confidence

Output:

- thesis-card revision cycle
- monthly opportunity board

## Quarterly Layer

Purpose:

- detect regime shifts
- update the ranking framework if needed
- compare reality against prior forecasts

Output:

- quarterly strategy review
- methodology adjustment memo

## Rule

The slower layers should never become disconnected from the faster layers.

The point is compounding judgment, not just compounding raw data.
