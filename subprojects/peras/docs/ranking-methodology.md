# Ranking Methodology

## Objective

Rank AI directions by future value density, not by present attention.

## Core Principle

Peras should reward directions that show:

- real technical movement
- credible buildability
- plausible commercialization
- underappreciated upside

It should penalize directions that show:

- weak evidence
- low implementation plausibility
- crowding without moat
- narrative inflation unsupported by frontier movement

## First Ranking Dimensions

Each direction should be scored on a simple first-pass scale such as `1-5`.

### 1. Frontier Movement

Signals:

- strong paper velocity
- citations or review attention
- top conference visibility
- frontier-lab involvement

### 2. Reproducible Build Momentum

Signals:

- open-source activity
- implementation quality
- tooling and framework support
- signs of repeatable engineering rather than one-off demos

### 3. Commercial Pull

Signals:

- enterprise demand
- company launches
- credible customer pain
- industrial integration paths

### 4. Time-To-Product

Question:

how far is the direction from becoming a product that a serious team could ship?

### 5. Value Asymmetry

Question:

if this direction succeeds, how large is the upside relative to current market attention?

### 6. Durability

Question:

does the direction have a path to defensible long-term value, or is it likely to be commoditized quickly?

### 7. Bottleneck And Fragility

Signals:

- data bottlenecks
- compute bottlenecks
- regulatory bottlenecks
- integration bottlenecks
- evaluation fragility

## Initial Ranking Logic

The first version should not hide everything behind one number.

Use:

- a headline rank
- dimension-level sub-scores
- confidence band
- explicit evidence notes

## Uncertainty Rule

Every rank should state:

- what is known
- what is inferred
- what is still speculative
