# Source Architecture

Peras should combine four source classes.

## 1. Academic Frontier

Use for:

- frontier capability movement
- research momentum
- methodological novelty

Examples:

- arXiv
- OpenAlex
- Semantic Scholar
- Crossref

## 2. Open Technical Build Surface

Use for:

- open-source implementation velocity
- repo emergence
- toolchain adoption

Examples:

- GitHub Search / repository APIs
- Papers with Code
- Hugging Face papers / model ecosystem surfaces

## 3. Business / Industry Movement

Use for:

- commercialization
- deployment intent
- enterprise demand signals

Examples:

- SEC EDGAR
- official company blogs / product release feeds
- major cloud / platform AI launch feeds

## 4. Historical Pattern Memory

Use for:

- comparing current AI waves to prior technological transitions
- avoiding recency bias
- spotting repeated structure in how value actually compounds

This layer should not become lazy analogy-making.

It should act as a disciplined comparison system.
